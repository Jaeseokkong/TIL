# 🏗️ Spring 계층 구조 정리 

## 1️⃣ 구조 개요
```markdown
Controller → Service → DAO/Repository → Entity
       ↕
     DTO (입출력용)
```

|계층|역할|
|Controller|클라이언트 요청 처리, DTO 변환, 결과 반환|
|Service|비지니스 로직, 검증, Entity 생성/업데이트|
|DAO / Respository|DB 접근, Entity CRUD 수행|
|Entity|DB 테이블과 매핑되는 객체, 영속성 관리|
|DTO|데이터 전송 객체, API 요청/응답 용도|
