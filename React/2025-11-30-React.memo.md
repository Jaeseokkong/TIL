# React.memo
`React.memo`는 **컴포넌트의 불필요한 리렌더링을 방지하는 고차 컴포넌트(HOC, Higher-Order Component)** 로, `props`가 바뀌지 않았다면 기존 렌더링 결과를 재사용하여 성능을 개선합니다.

---

## 1️⃣ React.memo를 사용하는 이유

- 부모가 렌더링될 때 **자식도 매번 리렌더링되는 문제 해결**
- **리렌더링 비용이 큰 컴포넌트 최적화**
- 동일한 UI를 다시 계산하지 않도록 **렌더링 결과 재사용**

---


## 2️⃣ React.memo 기본 사용 예시

### 🧐 예제 1: React.memo 적용 전

```jsx
function Child({ count }: { count: number }) {
  console.log("Child 렌더링!");
  return <p>Count: {count}</p>;
}
```

부모가 텍스트 입력 변경 등으로 렌더링될 때마다 자식도 매번 렌더링됨.

### 🧐 예제 2: React.memo 적용 후

```jsx
const Child = React.memo(({ count }: { count: number }) => {
  console.log("Child 렌더링!");
  return <p>Count: {count}</p>;
});
```

✔ `count`가 바뀌지 않는 한 렌더링 안 함.
